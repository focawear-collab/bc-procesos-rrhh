TECHNICAL SPECIFICATIONS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Architecture:
  • Frontend: Single-page HTML with inline CSS + JavaScript (no bundler)
  • Backend: Vercel serverless functions (Node.js)
  • Database: Notion API integration
  • Deployment: Vercel (static + serverless hybrid)

API Design:
  • RESTful endpoints (POST, GET, PATCH)
  • JSON request/response bodies
  • CORS enabled
  • 500ms debounce on auto-save

Database:
  • 1 Notion database (10 properties)
  • 59 checklist items (distributed across 3 tracks)
  • JSON array storage for checked items
  • Unique ID per proceso (RC prefix)

Performance:
  • Page load: <1s (static HTML)
  • API latency: 100-300ms (Notion dependent)
  • Bundle size: ~65 KB HTML (uncompressed), ~4 KB gzipped
  • Checkbox save: 500ms debounce (UX-optimized)

Browser Support:
  • Chrome/Edge 90+
  • Firefox 88+
  • Safari 14+
  • Mobile browsers (responsive)

GETTING STARTED
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Quick Start (15 minutes):

1. Read DEPLOYMENT.md for step-by-step instructions

2. Prepare Notion:
   • Visit https://www.notion.com/my-integrations
   • Create integration (or use existing)
   • Copy secret token
   • Share database 37c3e919a705410cae2430e8f33b99fc with integration

3. Deploy to Vercel:
   • Push to GitHub (or import without git)
   • Go to https://vercel.com/dashboard
   • Add project from GitHub
   • Set environment variables:
     - NOTION_TOKEN = [token from step 2]
     - NOTION_DATABASE_ID = 37c3e919a705410cae2430e8f33b99fc
   • Deploy

4. Test:
   • Visit deployment URL
   • Click "+ Nuevo Proceso"
   • Create test proceso
   • Check boxes
   • Refresh → should persist (Notion sync proof!)

For detailed instructions, see DEPLOYMENT.md.
