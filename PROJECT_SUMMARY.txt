TECHNICAL ARCHITECTURE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

FRONTEND (HTML/CSS/JavaScript)
  • Single-file architecture: CSS + JS inline in HTML (no bundler needed)
  • Design system: Matches bc-anonimo.vercel.app (dark theme, gold accents)
  • Responsive: Mobile-first, tested at 375px+ breakpoints
  • Interactions:
    - Tab switching (4 tabs)
    - Checkbox state management with visual feedback
    - Modal form with validation
    - Toast notifications for user feedback
    - Progress bars with real-time updates

BACKEND (Serverless Functions)
  • Framework: Vercel serverless (native Node.js)
  • API: Native fetch() calls to Notion API
  • Pattern: Handler functions receiving (req, res)
  • CORS: Enabled on all endpoints for cross-origin requests
  • Error handling: Try/catch with JSON error responses

DATABASE (Notion)
  • Integration: Native Notion API (v2022-06-28)
  • Authentication: Bearer token (environment variable)
  • Storage: Single database with 10 properties
  • Persistence: No localStorage — all state in Notion
  • Sync: Debounced (500ms) auto-save on checkbox changes

DEPLOYMENT
  • Platform: Vercel (Edge Functions + Static Files)
  • Build: Minimal (no build step needed)
  • Environment: Production-ready with secret management
  • Domain: Vercel auto-domain (custom domain optional)

API SPECIFICATION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

POST /api/procesos
  Create new recruitment process
  
  Request body:
  {
    "candidato": "Juan Pérez",
    "cargo": "Garzón/a",
    "local": "BC1",
    "notas": "Referencia de Felipe..."  // optional
  }
  
  Response:
  {
    "id": "page-id-uuid",
    "candidato": "Juan Pérez",
    "cargo": "Garzón/a",
    "local": "BC1",
    "fase": "F1 Captación",
    "estado": "En curso",
    "avance": 0,
    "fechaInicio": "2026-04-09",
    "items_completados": [],
    "notas": "Referencia de Felipe...",
    "idProceso": "RC1234"
  }

GET /api/procesos?status=[active|all|completed]
  Fetch procesos (default: active only)
  
  Response: Array of proceso objects (same schema as POST)

PATCH /api/proceso?id=<page_id>
  Update specific proceso
  
  Request body (all optional):
  {
    "fase": "F2 Entrevista",
    "estado": "En curso",
    "avance": 45,
    "items_completados": "[\"garzon-f1-0\",\"garzon-f1-1\"]",
    "notas": "Updated notes"
  }
  
  Response: Updated proceso object

GET /api/health
  Health check
  
  Response:
  {
    "ok": true,
    "timestamp": "2026-04-09T12:34:56.789Z"
  }

DEPLOYMENT INSTRUCTIONS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

QUICK START (15 minutes)

1. Prepare Notion
   • Go to https://www.notion.com/my-integrations
   • Create integration (or use existing)
   • Copy secret token
   • Share database 37c3e919a705410cae2430e8f33b99fc with integration

2. Push to GitHub
   git init
   git add .
   git commit -m "Initial: HR recruitment platform"
   git remote add origin https://github.com/yourname/bc-procesos-rrhh
   git push -u origin main

3. Deploy to Vercel
   • Go to https://vercel.com/dashboard
   • Click "Add New" → "Project"
   • Select your GitHub repo
   • Add environment variables:
     - NOTION_TOKEN = [paste token]
     - NOTION_DATABASE_ID = 37c3e919a705410cae2430e8f33b99fc
   • Click "Deploy"

4. Verify
   • Wait 2-3 min for build
   • Visit https://bc-procesos-rrhh.vercel.app
   • Test: Click "+ Nuevo Proceso"
   • Create test proceso
   • Check boxes
   • Refresh → checkboxes persist (proof: Notion sync works!)

See DEPLOYMENT.md for detailed step-by-step guide.
